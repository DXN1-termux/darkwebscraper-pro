## 🧠 THE DEDUPLICATION ENGINE — ALL 10 LAYERS EXPLAINED

> This is the crown jewel. A **10-layer deduplication gauntlet** that a link has to survive before it's considered "new". Most scrapers check one or two things. This checks **ten**, in order from cheapest to most expensive. If a link fails any layer, it's dead.

### 📐 Layer-by-Layer Technical Reference

| # | Layer Name | Algorithm | Data Structure | Cost |
|:---:|:---|:---|:---|:---:|
| 1 | **Exact Content Hash** | MD5 of `title.lower() \| normalize(url)` | `set` | 🟢 O(1) |
| 2 | **Domain Duplicate** | Regex extract `.onion` 16/56-char hash | `set` | 🟢 O(1) |
| 3 | **Normalized URL** | Strip tracking params + sort remaining | `set` | 🟢 O(1) |
| 4 | **Path Signature** | Lowercase + collapse `//` + `\d+→{num}` | `set` | 🟢 O(1) |
| 5 | **Content Fingerprint** | Domain + top-5 non-stopword title tokens | `set` (MD5) | 🟡 O(n words) |
| 6 | **Phonetic Hash** | Soundex-style consonant group mapping | `set` | 🟡 O(n chars) |
| 7 | **Canonical URL** | Strip index/home/default + `{ver}` + `{id}` | `set` | 🟡 O(path len) |
| 8 | **Bigram Word Hash** | MD5 of consecutive significant word pairs | `set` (overlap ratio) | 🟠 O(n²) |
| 9 | **Fuzzy Title** | Jaccard on word sets + SequenceMatcher | `list` (linear scan) | 🔴 O(n·m) |
| 10 | **Fuzzy URL** | Same netloc + SequenceMatcher on path | `list` (linear scan) | 🔴 O(n·m) |

> **Layers 1–7** are O(1) set lookups after pre-computation. **Layers 8–10** are expensive and only run if a link survives all previous checks. This ordering ensures maximum speed with maximum coverage.

### 🧪 Real-World Dedup Examples

```python
# Layer 2 — Domain catches alternate paths to same .onion
"http://exampledomain56chars.onion/"   ← SEEN
"http://exampledomain56chars.onion/login"  → 💀 DUPLICATE (same domain)

# Layer 3 — Normalized URL strips tracking junk
"http://site.onion/page?ref=telegram&id=99"
"http://site.onion/page?id=99"         → 💀 DUPLICATE (normalized match)

# Layer 4 — Path signature collapses numbers
"http://shop.onion/item/4829"  ← SEEN
"http://shop.onion/item/9173"  → 💀 DUPLICATE (path sig: /item/{num})

# Layer 6 — Phonetic catches typo variants
"Empire Market"  ← SEEN title
"Empyre Market"  → 💀 DUPLICATE (same phonetic hash E516)

# Layer 7 — Canonical strips version/index noise
"http://site.onion/v2.1/index.php"  ← SEEN
"http://site.onion/v3.0/"          → 💀 DUPLICATE (canonical: site.onion)

# Layer 9 — Fuzzy title catches rewording
"The Hidden Wiki – Updated 2026 Edition"  ← SEEN
"Hidden Wiki 2026 (Updated)"              → 💀 DUPLICATE (Jaccard > 0.85)

# Layer 10 — Fuzzy URL catches param variations
"http://market.onion/listing?id=1&cat=drugs"  ← SEEN
"http://market.onion/listing?id=2&cat=drugs"  → 💀 DUPLICATE (same keys)
```

## ⚡ Parallel Processing Architecture

```
                     ┌───────────────────────────┐
                     │     entries[] — N items    │
                     └────────────┬──────────────┘
                                  │ split into chunks
              ┌────────┬──────────┼──────────┬────────┐
              ▼        ▼          ▼          ▼        ▼
          [chunk 0] [chunk 1] [chunk 2] [chunk 3] [chunk 4]
              │        │          │          │        │
       ┌──────▼──┐ ┌───▼─────┐ ┌─▼──────┐ ... ...  │
       │Worker 0 │ │Worker 1 │ │Worker 2 │          │
       │DedupEng │ │DedupEng │ │DedupEng │          │
       │(no load)│ │(no load)│ │(no load)│          │
       └──────┬──┘ └───┬─────┘ └─┬──────┘          │
              └────────┴──────────┴─────────────────┘
                                  │
                     ┌────────────▼───────────────┐
                     │  SORT results by chunk_idx  │
                     │  MERGE into final_engine    │
                     │  Cross-chunk dedup pass      │
                     │  (catches inter-chunk dups)  │
                     └────────────┬───────────────┘
                                  │
                     ┌────────────▼───────────────┐
                     │  ✅ Final unique entries[]  │
                     └───────────────────────────┘

  Threshold: ≥50 entries → parallel. <50 → single engine (faster for small sets)
  Workers: min(max(4, os.cpu_count()), 6) — always right-sized
```

## 💾 Persistent State — 8 State Vectors

Every time a new link is accepted, it's added to **8 separate state vectors** in `seen_data.json`:

```json
{
  "hashes":      ["md5hex...", ...],   // Layer 1 + 5: exact hash + content fingerprint
  "domains":     ["abc123.onion", ...],// Layer 2: base .onion domain
  "paths":       ["/item/{num}", ...], // Layer 4: path signatures
  "titles":      ["Full Title...", ...],// Layer 9: fuzzy title list
  "links":       ["http://...", ...],  // Layer 3 + 10: normalized URLs
  "phonetic":    ["H516", ...],        // Layer 6: Soundex-style hashes
  "canonical":   ["site.onion/...", ...],// Layer 7: canonical URLs
  "title_hashes":["a1b2c3d4", ...]    // Layer 8: bigram word hash pool
}
```

> This file is loaded at startup and saved after every single accepted entry. **You can kill the process mid-run and resume with zero data loss.**
